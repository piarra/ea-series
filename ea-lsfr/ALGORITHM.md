XAUUSD ストップ狩り逆張りアルゴリズム仕様

ファイル名: LSFR_v1.mq5

1. コンセプト

本アルゴリズムの目的は、XAUUSDにおいて発生するストップ狩り的な価格の一時的な流動性掃除を検出し、その後の失敗したブレイクアウトに対して逆張りを行うことで、短期的な価格回帰を収益化することである。

本アルゴリズムは、将来のストップ狩りを完全に予測することを目指すものではない。
代わりに、以下の一連の市場行動を条件付きで捉える。
	1.	市場参加者のストップ注文が溜まりやすい価格帯を事前に定義する
	2.	価格がその水準を一時的に突破する動き（sweep）を検出する
	3.	その突破が継続せず、元のレンジ内へ速やかに回帰する失敗を確認する
	4.	失敗が確認された時点で、その突破方向とは逆向きにエントリーする

したがって、本戦略の本質は予測型アルゴリズムではなく、流動性イベント後の失敗を利用する反応型アルゴリズムである。

⸻

2. 戦略目的
	•	XAUUSDにおける短期的な流動性掃除の後に起こる価格回帰を取る
	•	本物のブレイクアウトではなく、フェイクブレイクのみを対象とする
	•	高頻度売買ではなく、明確な条件が整った局面のみ執行する選別型ロジックとする
	•	ニュースドリブンな価格再評価と、単なる流動性狩りを区別する

3. 対象市場・前提

3.1 対象銘柄
	•	XAUUSD（スポット金）

3.2 補助参照市場
	•	CME Gold Futures（GC）

3.3 想定時間軸
	•	シグナル検出: 1分足
	•	補助判断: 5分足、1時間足
	•	上位トレンド判定: 1時間足または4時間足

3.4 売買スタイル
	•	短期逆張り
	•	セッション内完結を基本
	•	保有時間は数分〜数十分を想定

4. 基本戦略ロジック

本アルゴリズムは以下の4段階で構成される。

4.1 流動性プール候補の特定

価格がストップ注文を集めやすい水準を事前に抽出する。

4.2 Sweepの検出

価格が当該水準を一時的に突破したことを検出する。

4.3 失敗ブレイクの確認

突破後、価格が継続できず水準内へ戻ることを確認する。

4.4 逆張り執行

失敗が確認された場合、突破方向とは逆方向にポジションを持つ。

⸻

5. 流動性レベル定義

ストップが溜まりやすい価格帯として、以下を対象とする。

5.1 直近高値・安値
	•	過去N本の局所高値
	•	過去N本の局所安値

5.2 セッション基準値
	•	前日高値 / 前日安値
	•	アジア時間高値 / 安値
	•	ロンドン時間高値 / 安値
	•	ニューヨーク時間高値 / 安値

5.3 ラウンドナンバー
	•	00, 50 を中心とする節目
	•	必要に応じて 25, 75 も候補に含める

例:
	•	3000
	•	3050
	•	3075
	•	3100

5.4 価格滞留・平均回帰基準
	•	セッションVWAP
	•	前日VWAP
	•	VWAPからの乖離帯
	•	前日高出来高帯の外縁

5.5 補助参照レベル
	•	GC先物で明確に意識された直近高値 / 安値
	•	スポットと先物の両方で重なる価格帯

6. Sweep検出仕様

6.1 定義

Sweepとは、価格が重要レベルを明確に突破するが、その後継続しない一時的な流動性掃除を指す。

6.2 上方向Sweep

以下を満たす場合、上方向のストップ狩り候補とする。
	•	high_t > L + epsilon

ここで
	•	L = 対象流動性レベル
	•	epsilon = ノイズ除去幅

6.3 下方向Sweep

以下を満たす場合、下方向のストップ狩り候補とする。
	•	low_t < L - epsilon

6.4 ノイズ除去

epsilon は固定値ではなくATR連動とする。

例:
	•	epsilon = 0.08 × ATR_1m

⸻

7. 失敗ブレイク判定仕様

本アルゴリズムでは、単なる突破ではエントリーしない。
必ず突破失敗を確認する。

7.1 レベル回帰条件

上方向sweep後:
	•	close_t < L または
	•	close_t+1 < L

下方向sweep後:
	•	close_t > L または
	•	close_t+1 > L

7.2 ヒゲ比率条件

上抜け失敗:
	•	upper_wick / candle_range > threshold

下抜け失敗:
	•	lower_wick / candle_range > threshold

初期推奨値:
	•	threshold = 0.55

7.3 フォロー失敗条件

sweep発生後の数本で、突破方向への進行効率が低い場合を失敗とみなす。

例:
	•	3本以内にブレイク方向への進展が小さい
	•	continuation efficiency が閾値未満

例:
	•	continuation_efficiency < 0.35

7.4 戻り速度条件

価格が一定時間内に水準内へ戻らない場合、その突破は本物である可能性が高いため除外する。

例:
	•	1〜3本以内に再侵入が必要

⸻

8. エントリー仕様

8.1 上方向sweep後のショート

以下をすべて満たした場合にショート候補とする。
	1.	重要高値 L を上抜けする
	2.	上抜け後、価格が L 未満へ戻る
	3.	wick比率またはフォロー失敗条件を満たす
	4.	フィルター条件に抵触しない

執行方法:
	•	reclaim確認後の成行ショート
	•	または L への戻りで指値ショート

8.2 下方向sweep後のロング

以下をすべて満たした場合にロング候補とする。
	1.	重要安値 L を下抜けする
	2.	下抜け後、価格が L 超へ戻る
	3.	wick比率またはフォロー失敗条件を満たす
	4.	フィルター条件に抵触しない

執行方法:
	•	reclaim確認後の成行ロング
	•	または L への戻りで指値ロング

9. 損切り仕様

損切りは、sweepで付けた極値のさらに外側に置く。

9.1 ショート時
	•	stop = sweep_high + buffer

9.2 ロング時
	•	stop = sweep_low - buffer

9.3 バッファ

バッファはATR連動とする。

例:
	•	buffer = 0.15 ～ 0.25 × ATR_1m

⸻

10. 利確仕様

利確は段階的に行う。

10.1 第1利確
	•	1R 到達
	•	または元レンジ中央

10.2 第2利確
	•	セッションVWAP
	•	セッションミッドポイント
	•	反対側の短期流動性帯

10.3 推奨運用
	•	50%を1Rで利確
	•	残りをVWAPまたはセッション平均回帰まで保持

⸻

11. フィルター仕様

本戦略の性能はフィルター品質に大きく依存する。

11.1 イベントフィルター

重要経済指標前後では新規エントリーを抑制する。

対象例:
	•	CPI
	•	NFP
	•	FOMC
	•	PPI
	•	ISM
	•	雇用関連指標

推奨ルール:
	•	発表前後 ±5〜15分は新規停止
	•	特に8:30 ET系指標は厳格に除外
	•	発表直後の第1波は見送り、第2失敗のみ対象とする選択も可

11.2 セッションフィルター

セッションごとに特性が異なるため、別管理とする。
	•	アジア時間
	•	ロンドン時間
	•	NY時間
	•	ロンドン-ニューヨーク重複時間

推奨:
	•	最適化はセッション別に実施
	•	執行許可時間を限定してもよい

11.3 トレンドフィルター

強い上位足トレンド中の逆張りを制限する。

例:
	•	1時間足EMA傾きが閾値以上なら逆張り禁止
	•	ADX高水準時は片側のみ許可
	•	1時間足で連続トレンド拡大型なら見送り

11.4 ボラティリティフィルター

異常な高ボラ・低流動性時を除外する。

例:
	•	ATR_1m / ATR_1h が異常値
	•	spread が閾値以上
	•	slippage想定が閾値以上

⸻

12. スコアリング仕様

各候補シグナルに対し、総合スコアを算出して執行可否を決める。

12.1 評価項目
	•	S1: レベル強度
	•	S2: sweep深さ
	•	S3: rejection強度
	•	S4: continuation失敗度
	•	S5: tick volume異常度
	•	S6: regime penalty

12.2 例

Score = 0.25*S1 + 0.20*S2 + 0.25*S3 + 0.15*S4 + 0.10*S5 - 0.25*S6

12.3 執行条件
	•	Score > THRESHOLD の場合のみ執行
	•	thresholdはバックテストで最適化

18. 実装上の原則
	•	予測よりも確認を重視する
	•	先回りエントリーを避ける
	•	本物ブレイクへの逆張りを減らすことを優先する
	•	指標・トレンド・流動性の3要素で無駄打ちを削減する
	•	シグナル数よりも期待値を重視する

⸻

19. 要約

本アルゴリズムは、XAUUSDにおいてストップ注文が集まりやすい価格帯を定義し、その価格帯を一時的に突破した後、突破が失敗してレンジ内へ戻る動きを検出して逆張りする戦略である。

中核となる考え方は、ストップ狩りそのものを事前予測することではなく、ストップ狩り後の「失敗したブレイク」を確認してから入ることにある。

そのため、戦略の優位性は以下に依存する。
	•	レベル定義の妥当性
	•	reclaim判定の精度
	•	強トレンド・指標時の除外
	•	執行コストを踏まえた実運用設計
